\subsection{2. Vulnerability Findings}\label{2-vulnerability-findings}

\subsubsection{2.1 HIGH SEVERITY}\label{21-high-severity}

\paragraph{H-01: Missing Reentrancy Guard in
Bridge.sol}\label{h-01-missing-reentrancy-guard-in-bridgesol}

\textbf{Location}:
\texttt{/Users/z/work/lux/standard/contracts/bridge/Bridge.sol}

\textbf{Description}: The Bridge contract handles token transfers and
vault deposits/withdrawals without ReentrancyGuard protection.

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{function vaultDeposit(uint256 amount\_, address tokenAddr\_) public payable \{}
\NormalTok{    if (tokenAddr\_ != address(0)) \{}
\NormalTok{        IERC20(tokenAddr\_).transferFrom(msg.sender, address(vault), amount\_);}
\NormalTok{    \}}
\NormalTok{    vault.deposit\{value: msg.value\}(tokenAddr\_, amount\_);}
\NormalTok{    emit VaultDeposit(msg.sender, amount\_, tokenAddr\_);}
\NormalTok{\}}
\end{Highlighting}
\end{Shaded}

\textbf{Impact}: Potential reentrancy via malicious ERC20 token with
callback hooks.

\textbf{Recommendation}:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Add \texttt{ReentrancyGuard} from OpenZeppelin
\item
  Apply \texttt{nonReentrant} modifier to all external functions
  involving transfers
\end{enumerate}

\begin{center}\rule{0.5\linewidth}{0.5pt}\end{center}

\paragraph{H-02: Missing Reentrancy Guard in
BridgeVault.sol}\label{h-02-missing-reentrancy-guard-in-bridgevaultsol}

\textbf{Location}:
\texttt{/Users/z/work/lux/standard/contracts/bridge/BridgeVault.sol}

\textbf{Description}: Vault operations lack reentrancy protection
despite handling ETH and ERC20 transfers.

\textbf{Impact}: Cross-function reentrancy attacks possible.

\subsection{7. Conclusion}\label{7-conclusion}

The Lux smart contract ecosystem demonstrates modern Solidity practices
including:

\begin{itemize}
\tightlist
\item
  Use of OpenZeppelin contracts for standard patterns
\item
  EIP-7201 namespaced storage for upgradeability
\item
  Proper access control hierarchies
\item
  Well-structured precompile interfaces
\end{itemize}

\textbf{Critical gaps} that require immediate remediation:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Reentrancy protection in Bridge and Router contracts
\item
  Circuit breaker mechanisms for high-leverage perpetuals
\item
  Timelock delays for governance parameter changes
\end{enumerate}

The FHE integration is well-designed with proper precompile abstraction,
though the main FHE.sol file would benefit from modularization for
maintainability.
