\section{Gas Optimization and DoS Attack Vector
Audit}\label{gas-optimization-and-dos-attack-vector-audit}

\subsection{Executive Summary}\label{executive-summary}

This audit analyzes the Lux Standard contracts for gas optimization
opportunities and Denial of Service (DoS) attack vectors. Several
critical and high-severity issues were identified, primarily involving
unbounded loops that iterate over user-controlled array sizes.

\textbf{Risk Summary:}

\begin{itemize}
\tightlist
\item
  Critical: 4 issues
\item
  High: 5 issues
\item
  Medium: 8 issues
\item
  Low/Informational: 6 issues
\end{itemize}

\begin{center}\rule{0.5\linewidth}{0.5pt}\end{center}

\subsection{Critical Severity}\label{critical-severity}

\subsubsection{\texorpdfstring{C-01: Unbounded Loop in
\texttt{Bond.getActiveBonds()}}{C-01: Unbounded Loop in Bond.getActiveBonds()}}\label{c-01-unbounded-loop-in-bondgetactivebonds}

\textbf{File:} \texttt{/contracts/treasury/Bond.sol} (Lines 227-240)

\textbf{Description:} The \texttt{getActiveBonds()} function iterates
twice over all bonds ever created, without any bounds:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{function getActiveBonds() external view returns (uint256[] memory ids) \{}
\NormalTok{    uint256 count = 0;}
\NormalTok{    for (uint256 i = 0; i \textless{} nextBondId; i++) \{      // First loop}
\NormalTok{        if (bonds[i].active) count++;}
\NormalTok{    \}}

\NormalTok{    ids = new uint256[](count);}
\NormalTok{    uint256 j = 0;}
\NormalTok{    for (uint256 i = 0; i \textless{} nextBondId; i++) \{      // Second loop}
\NormalTok{        if (bonds[i].active) \{}
\NormalTok{            ids[j++] = i;}
\NormalTok{        \}}
\NormalTok{    \}}
\NormalTok{\}}
\end{Highlighting}
\end{Shaded}

\textbf{Attack Vector:}

\begin{itemize}
\tightlist
\item
  An attacker with owner privileges (or compromised owner) can create
  thousands of bonds
\item
  Even if bonds are closed, they remain in storage and must be iterated
\item
  Each iteration performs an SLOAD (\textasciitilde2100 gas cold,
  \textasciitilde100 gas warm)
\item
  With 1000 bonds: \textasciitilde2.1M gas minimum, potentially
  exceeding block gas limit
\end{itemize}

\textbf{Impact:} Complete DoS of view function; any off-chain
integrations relying on this function will fail

\textbf{Recommendation:}

\begin{itemize}
\tightlist
\item
  Maintain a separate array of active bond IDs
\item
  Implement pagination:
  \texttt{getActiveBonds(uint256\ offset,\ uint256\ limit)}
\item
  Use an EnumerableSet for active bonds
\end{itemize}

\begin{center}\rule{0.5\linewidth}{0.5pt}\end{center}

\subsubsection{\texorpdfstring{C-02: Unbounded Loop in
\texttt{ProtocolLiquidity.claimAll()}}{C-02: Unbounded Loop in ProtocolLiquidity.claimAll()}}\label{c-02-unbounded-loop-in-protocolliquidityclaimall}

\textbf{File:} \texttt{/contracts/treasury/ProtocolLiquidity.sol} (Lines
360-376)

\textbf{Description:}

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{function claimAll() external nonReentrant \{}
\NormalTok{    uint256 total = 0;}
\NormalTok{    uint256 count = userPositionCount[msg.sender];}

\NormalTok{    for (uint256 i = 0; i \textless{} count; i++) \{           // Unbounded loop}
\NormalTok{        VestingPosition storage pos = positions[msg.sender][i];}
\NormalTok{        uint256 claimable = \_claimable(pos);}
\NormalTok{        if (claimable \textgreater{} 0) \{}
\NormalTok{            pos.claimed += claimable;               // SSTORE per position}
\NormalTok{            total += claimable;}
\NormalTok{            emit Claimed(msg.sender, i, claimable); // Event per position}
\NormalTok{        \}}
\NormalTok{    \}}
\NormalTok{    // ...}
\NormalTok{\}}
\end{Highlighting}
\end{Shaded}

\textbf{Attack Vector:}

\begin{itemize}
\tightlist
\item
  User creates many small positions via \texttt{bondLP()} or
  \texttt{depositSingleSided()}
\item
  Each iteration: \textasciitilde5000 gas (SLOAD) + \textasciitilde20000
  gas (SSTORE if claimable) + \textasciitilde2000 gas (event)
\item
  200 positions = \textasciitilde5.4M gas, exceeding most block gas
  limits
\item
  User\textquotesingle s funds become permanently locked as
  \texttt{claimAll()} always fails
\end{itemize}

\textbf{Impact:} Permanent fund lockup for affected users

\textbf{Recommendation:}

\begin{itemize}
\tightlist
\item
  Implement batch claiming with pagination:
  \texttt{claimBatch(uint256{[}{]}\ calldata\ positionIds)}
\item
  Add maximum positions per user limit
\item
  Allow individual position claims (already exists via
  \texttt{claim(positionId)})
\end{itemize}

\subsection{State Bloat Attack
Vectors}\label{state-bloat-attack-vectors}

\subsubsection{SB-01: Unlimited Position
Creation}\label{sb-01-unlimited-position-creation}

\textbf{Files:} \texttt{ProtocolLiquidity.sol}, \texttt{LiquidBond.sol}

Users can create unlimited vesting positions with minimum amounts. Each
position consumes storage slots:

\begin{itemize}
\tightlist
\item
  \texttt{VestingPosition}: 4 slots (\textasciitilde80,000 gas to write)
\item
  \texttt{Purchase}: 7 slots (\textasciitilde140,000 gas to write)
\end{itemize}

\textbf{Attack Cost:} With small minimum bonds, attacker can bloat state
cheaply.

\textbf{Recommendation:}

\begin{itemize}
\tightlist
\item
  Enforce meaningful minimum deposit amounts
\item
  Limit positions per user
\item
  Consider position consolidation mechanism
\end{itemize}

\begin{center}\rule{0.5\linewidth}{0.5pt}\end{center}

\subsubsection{SB-02: Unlimited Bond/Pool Creation
(Admin)}\label{sb-02-unlimited-bondpool-creation-admin}

\textbf{Files:} \texttt{Bond.sol}, \texttt{ProtocolLiquidity.sol}

Compromised admin can create unlimited bonds/pools, bloating state and
making iteration-based functions fail.

\textbf{Recommendation:} Add maximum limits for total bonds/pools

\begin{Shaded}

\subsubsection{GO-02: Use Unchecked Math Where
Safe}\label{go-02-use-unchecked-math-where-safe}

\textbf{Example:} Loop counters with known bounds:

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{for (uint256 i = 0; i \textless{} length;) \{}
\NormalTok{    // ...}
\NormalTok{    unchecked \{ ++i; \}}
\NormalTok{\}}
\end{Highlighting}
\end{Shaded}

Saves \textasciitilde50 gas per iteration.

\subsubsection{GO-03: Cache Array
Lengths}\label{go-03-cache-array-lengths}

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{// Before}
\NormalTok{for (uint256 i = 0; i \textless{} tokens.length; i++) \{ ... \}}

\NormalTok{// After}
\NormalTok{uint256 len = tokens.length;}
\NormalTok{for (uint256 i = 0; i \textless{} len; i++) \{ ... \}}
\end{Highlighting}
\end{Shaded}

Saves \textasciitilde3 gas per iteration.

\subsection{Recommendations Summary}\label{recommendations-summary}

\subsubsection{Immediate (Critical):}\label{immediate-critical}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Add pagination to all \texttt{get*} view functions
\item
  Implement \texttt{claimBatch(uint256{[}{]}\ positionIds)} pattern
\item
  Add maximum limits to user positions
\end{enumerate}

\subsubsection{Short-term (High):}\label{short-term-high}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Add \texttt{MAX\_BATCH\_SIZE} constants to all batch functions
\item
  Implement lazy weight calculation in GaugeController
\item
  Add explicit array length validation
\end{enumerate}

\subsubsection{Medium-term:}\label{medium-term}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Optimize struct packing for storage efficiency
\item
  Use unchecked math in bounded loops
\item
  Consider EnumerableSet for active items
\item
  Emit summary events instead of per-item events
\end{enumerate}

\subsubsection{Architecture:}\label{architecture}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Consider off-chain indexing for complex queries
\item
  Implement pull-over-push patterns where applicable
\item
  Add circuit breakers for gas-heavy operations
\end{enumerate}
