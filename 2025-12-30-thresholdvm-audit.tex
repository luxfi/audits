\subsection{Executive Summary}\label{executive-summary}

ThresholdVM implements a threshold FHE (Fully Homomorphic Encryption)
and MPC (Multi-Party Computation) chain for the Lux blockchain
ecosystem. The implementation supports multiple cryptographic protocols
including LSS, CGGMP21/CMP, BLS, FROST, EdDSA, and Ringtail
(post-quantum).

\subsubsection{\texorpdfstring{Security Posture:
\textbf{MODERATE}}{Security Posture: MODERATE}}\label{security-posture-moderate}

The codebase demonstrates sound cryptographic design with proper
protocol implementations. However, several areas require attention
before production deployment.

Implementation}\label{11-scheme-implementation}

\textbf{Findings}:

The implementation uses CKKS (Cheon-Kim-Kim-Song) scheme exclusively via
the lattice library\textquotesingle s E2S (Encryption-to-Share)
multiparty protocol.

\textbf{Observation}: TFHE and BGV schemes mentioned in audit scope are
NOT implemented. Only CKKS is present.

\textbf{FINDING-FHE-001} (Medium): Missing TFHE/BGV Support

\begin{itemize}
\tightlist
\item
  The audit scope specified TFHE, CKKS, and BGV schemes
\item
  Only CKKS is implemented
\item
  \textbf{Risk}: Limited FHE flexibility; TFHE preferred for boolean
  circuits
\item
  \textbf{Recommendation}: Document supported schemes clearly; consider
  TFHE for binary operations
\end{itemize}

needs chain ID binding to prevent cross-chain replay attacks.
